Section 7.3 Equivalence Classes

Worksheet Beginning Activity 1: Sets Associated with a Relation
A4US

As was indicated in Section 7.2, an equivalence relation on a set \(A\) is a relation with a certain combination of properties (reflexive, symmetric, and transitive) that allow us to sort the elements of the set into certain classes. This is done by means of certain subsets of \(A\) that are associated with the e lements of the set \(A\text{.}\) This will be illustrated with the following example. Let \(A = \left\{ {a, b, c, d, e} \right\}\text{,}\) and let \(R\) be the relation on the set \(A\) defined as follows:

\begin{align*} a \amp \mathrel{R} a \amp b \amp \mathrel{R} b \amp c \amp \mathrel{R} c \amp d \amp \mathrel{R} d \amp e \amp \mathrel{R} e\\ a \amp \mathrel{R} b \amp b \amp \mathrel{R} a \amp b \amp \mathrel{R} e \amp e \amp \mathrel{R} b\\ a \amp \mathrel{R} e \amp e \amp \mathrel{R} a \amp c \amp \mathrel{R} d \amp d \amp \mathrel{R} c \end{align*}

For each \(y \in A\text{,}\) define the subset \(R[y]\) of \(A\) as follows:

\begin{equation*} R[y] = \left\{ x \in A \mid x \mathrel{R} y \right\}\text{.} \end{equation*}

That is, \(R[y]\) consists of those elements in \(A\) such that \(x \mathrel{R} y\text{.}\) For example, using \(y = a\text{,}\) we see that \(a \mathrel{R} a\text{,}\) \(b \mathrel{R} a\text{,}\) and \(e \mathrel{R} a\text{,}\) and so \(R[a] = \{ a, b, e \}\text{.}\)

2.

3.

Which of the sets \(R[ a ]\text{,}\) \(R[ b ]\text{,}\) \(R[ c ]\text{,}\) \(R[ d ]\text{,}\) and \(R[ e ]\) are equal?

4.

Which of the sets \(R[ a ]\text{,}\) \(R[ b ]\text{,}\) \(R[ c ]\text{,}\) \(R[ d ]\text{,}\) and \(R[ e ]\) are disjoint?

As we will see in this section, the relationships between these sets are typical for an equivalence relation. The following example will show how different this can be for a relation that is not an equivalence relation.

Let \(A = \left\{ {a, b, c, d, e} \right\}\text{,}\) and let \(S\) be the relation on the set \(A\) defined as follows:

\begin{align*} b \amp \mathrel{S} b \amp c \amp \mathrel{S} c \amp d \amp \mathrel{S} d \amp e \amp \mathrel{S} e\\ a \amp \mathrel{S} b \amp a \amp \mathrel{S} d \amp b \amp \mathrel{S} c\\ c \amp \mathrel{S} d \amp d \amp \mathrel{S} c \end{align*}

5.

Draw a digraph that represents the relation \(S\) on \(A\text{.}\) Explain why \(S\) is not an equivalence relation on \(A\text{.}\)

For each \(y \in A\text{,}\) define the subset \(S[y]\) of \(A\) as follows:

\begin{equation*} S[ y ] = \left\{ { {x \in A } \mid x \mathrel{S} y} \right\} = \left\{ { {x \in A } \mid \left( {x, y} \right) \in S} \right\}\text{.} \end{equation*}

For example, using \(y = b\text{,}\) we see that \(S[b] = \{ a, b \}\) since \((a, b) \in S\) and \((b, b) \in S\text{.}\) In addition, we see that \(S[a] = \emptyset\) since there is no \(x \in A\) such that \((x, a) \in S\text{.}\)

Which of the sets \(S[ a ]\text{,}\) \(S[ b ]\text{,}\) \(S[ c ]\text{,}\) \(S[ d ]\text{,}\) and \(S[ e ]\) are equal?

8.

Which of the sets \(S[ b ]\text{,}\) \(S[ c ]\text{,}\) \(S[ d ]\text{,}\) and \(S[ e ]\) are disjoint?

Subsection The Definition of an Equivalence Class

We have indicated that an equivalence relation on a set is a relation with a certain combination of properties (reflexive, symmetric, and transitive) that allow us to sort the elements of the set into certain classes. We saw this happen in the beginning activities. We can now illustrate specifically what this means. For example, in Beginning Activity 2 , we used the equivalence relation of congruence modulo 3 on \(\mathbb{Z}\) to construct the following three sets:

\begin{align*} C[ 0 ] \amp = \left\{ { {a \in \mathbb{Z} } \mid a \equiv 0 \pmod 3} \right\}\!,\\ C[ 1 ] \amp = \left\{ { {a \in \mathbb{Z} } \mid a \equiv 1 \pmod 3} \right\}\!,\text{ and }\\ C[ 2 ] \amp = \left\{ { {a \in \mathbb{Z} } \mid a \equiv 2 \pmod 3} \right\}\!\text{.} \end{align*}

The main results that we want to use now are Theorem 3.5.9 and Corollary 3.5.10. This corollary tells us that for any \(a \in \mathbb{Z}\text{,}\) \(a\) is congruent to precisely one of the integers 0, 1, or 2. Consequently, the integer \(a\) must be congruent to 0, 1, or 2, and it cannot be congruent to two of these numbers. Thus

  1. For each \(a \in \mathbb{Z}\text{,}\) \(a \in C[ 0 ]\text{,}\) \(a \in C[ 1 ]\text{,}\) or \(a \in C[ 2 ]\text{;}\) and

  2. \(C[ 0 ] \cap C[ 1 ] = \emptyset\text{,}\) \(C[ 0 ] \cap C[ 2 ] = \emptyset\) , and \(C[ 1 ] \cap C[ 2 ] = \emptyset\text{.}\)

This means that the relation of congruence modulo 3 sorts the integers into three distinct sets, or classes, and that each pair of these sets have no elements in common. So if we use a rectangle to represent \(\mathbb{Z}\text{,}\) we can divide that rectangle into three smaller rectangles, corresponding to \(C[ 0 ]\text{,}\) \(C[ 1 ]\text{,}\) and \(C[ 2 ]\text{,}\) and we might picture this situation as follows:

Each integer is in exactly one of the three sets \(C[ 0 ]\text{,}\) \(C[ 1 ]\text{,}\) or \(C[ 2 ]\text{,}\) and two integers are congruent modulo 3 if and only if they are in the same set. We will see that, in a similar manner, if \(n\) is any natural number, then the relation of congruence modulo \(n\) can be used to sort the integers into \(n\) classes. We will also see that in general, if we have an equivalence relation \(R\) on a set \(A\text{,}\) we can sort the elements of the set \(A\) into classes in a similar manner.

Definition 7.3.1.

Let \(\sim\) be an equivalence relation on a nonempty set \(A\text{.}\) For each \(a \in A\text{,}\) the equivalence class of \(\boldsymbol{a}\) determined by \(\sim\) is the subset of \(A\text{,}\) denoted by \([ a ]\text{,}\) consisting of all the elements of \(A\) that are equivalent to \(a\text{.}\) That is,

\begin{equation*} [ a ] = \left\{ { {x \in A } \mid x \sim a} \right\}\!\text{.} \end{equation*}

We read \([ a ]\) as “the equivalence class of \(a\)” or as “bracket \(a\text{.}\)”

Subsubsection Notes

  1. We use the notation \([ a ]\) when only one equivalence relation is being used. If there is more than one equivalence relation, then we need to distinguish between the equivalence classes for each relation. We often use something like \([ a ]_{ \sim}\text{,}\) or if \(R\) is the name of the relation, we can use \(R[ a ]\) or \([a]_R\) for the equivalence class of \(a\) determined by \(R\text{.}\) In any case, always remember that when we are working with any equivalence relation on a set \(A\) if \(a \in A\text{,}\) then the equivalence class \([ a ]\) is a subset of \(A\text{.}\)

  2. We know that each integer has an equivalence class for the equivalence relation of congruence modulo 3. But as we have seen, there are really only three distinct equivalence classes. Using the notation from the definition, they are:

    \begin{align*} [0] \amp = \left\{ a \in \Z \mid \mod{a}{0}{3} \right\}, \amp [1] \amp = \left\{ a \in \Z \mid \mod{a}{1}{3} \right\}, \text{ and }\\ [2] \amp = \left\{ a \in \Z \mid \mod{a}{2}{3} \right\}\text{.} \end{align*}

Subsection Congruence Modulo \(\boldsymbol{n}\) and Congruence Classes

In Beginning Activity 2 , we used the notation \(C[ k ]\) for the set of all integers that are congruent to \(k\) modulo 3. We could have used a similar notation for equivalence classes, and this would have been perfectly acceptable. However, the notation \([ a ]\) is probably the most common notation for the equivalence class of \(a\text{.}\) We will now use this same notation when dealing with congruence modulo \(n\) when only one congruence relation is under consideration.

Definition 7.3.3.

Let \(n \in \mathbb{N}\text{.}\) Congruence modulo \(n\) is an equivalence relation on \(\mathbb{Z}\text{.}\) So for \(a \in \mathbb{Z}\text{,}\)

\begin{equation*} [ a ] = \left\{ { {x \in \mathbb{Z} } \mid x \equiv a \pmod n} \right\}\!\text{.} \end{equation*}

In this case, \([ a ]\) is called the congruence class of \(\boldsymbol{a}\) modulo \(\boldsymbol{n}\).

We have seen that congruence modulo 3 divides the integers into three distinct congruence classes. Each congruence class consists of those integers with the same remainder when divided by 3. In a similar manner, if we use congruence modulo 2, we simply divide the integers into two classes. One class will consist of all the integers that have a remainder of 0 when divided by 2, and the other class will consist of all the integers that have a remainder of 1 when divided by 2. That is, congruence modulo 2 simply divides the integers into the even and odd integers.

Subsection Properties of Equivalence Classes

As we have seen, in Beginning Activity 1 , the relation \(R\) was an equivalence relation. For that activity, we used \(R[ y ]\) to denote the equivalence class of \(y \in A\text{,}\) and we observed that these equivalence classes were either equal or disjoint.

However, in Beginning Activity 1 , the relation \(S\) was not an equivalence relation, and hence we do not use the term “equivalence class” for this relation. We should note, however, that the sets \(S[ y ]\) were not equal and were not disjoint. This exhibits one of the main distinctions between equivalence relations and relations that are not equivalence relations.

In Theorem 7.3.5, we will prove that if \(\sim\) is an equivalence relation on the set \(A\text{,}\) then we can “sort” the elements of \(A\) into distinct equivalence classes. The properties of equivalence classes that we will prove are as follows: (1) Every element of \(A\) is in its own equivalence class; (2) two elements are equivalent if and only if their equivalence classes are equal; and (3) two equivalence classes are either identical or they are disjoint.

Theorem 7.3.5.

Let \(A\) be a nonempty set and let \(\sim\) be an equivalence relation on the set \(A\text{.}\) Then,

  1. For each \(a \in A\text{,}\) \(a \in [ a ]\text{.}\)

  2. For each \(a, b \in A\text{,}\) \(a \sim b\) if and only if \([ a ] = [ b ]\text{.}\)

  3. For each \(a, b \in A\text{,}\) \([ a ] = [ b ]\) or \([ a ] \cap [ b ] = \emptyset\text{.}\)

Proof.

Let \(A\) be a nonempty set and assume that \(\sim\) is an equivalence relation on \(A\text{.}\) To prove the first part of the theorem, let \(a \in A\text{.}\) Since \(\sim\) is an equivalence relation on \(A\text{,}\) it is reflexive on \(A\text{.}\) Thus, \(a \sim a\text{,}\) and we can conclude that \(a \in [ a ]\text{.}\)

The second part of this theorem is a biconditional statement. We will prove it by proving two conditional statements. We will first prove that if \(a \sim b\text{,}\) then \([ a ] = [ b ]\text{.}\) So let \(a, b \in A\) and assume that \(a \sim b\text{.}\) We will now prove that the two sets \([ a ]\) and \([ b ]\) are equal. We will do this by proving that each is a subset of the other.

First, assume that \(x \in [ a ]\text{.}\) Then, by definition, \(x \sim a\text{.}\) Since we have assumed that \(a \sim b\text{,}\) we can use the transitive property of \(\sim\) to conclude that \(x \sim b\text{,}\) and this means that \(x \in [ b ]\text{.}\) This proves that \([ a ] \subseteq [ b ]\text{.}\)

We now assume that \(y \in [ b ]\text{.}\) This means that \(y \sim b\text{,}\) and hence by the symmetric property, that \(b \sim y\text{.}\) Again, we are assuming that \(a \sim b\text{.}\) So we have \(a \sim b\) and \(b \sim y\text{.}\) We use the transitive property to conclude that \(a \sim y\) and then, using the symmetric property, we conclude that \(y \sim a\text{.}\) This proves that \(y \in [ a ]\) and, hence, that \([ b ] \subseteq [ a ]\text{.}\) This means that we can conclude that if \(a \sim b\text{,}\) then \([ a ] = [ b ]\text{.}\)

We must now prove that if \([ a ] = [ b ]\text{,}\) then \(a \sim b\text{.}\) Let \(a, b \in A\) and assume that \([ a ] = [ b ]\text{.}\) Using the first part of the theorem, we know that \(a \in [ a ]\) and since the two sets are equal, this tells us that \(a \in [ b ]\text{.}\) Hence by the definition of \([ b ]\text{,}\) we conclude that \(a \sim b\text{.}\) This completes the proof of the second part of the theorem.

For the third part of the theorem, let \(a, b \in A\text{.}\) Since this part of the theorem is a disjunction, we will consider two cases: Either

\begin{equation*} [ a ] \cap [ b ] = \emptyset \text{ or } [ a ] \cap [ b ] \ne \emptyset\text{.} \end{equation*}

In the case where \([ a ] \cap [ b ] = \emptyset\text{,}\) the first part of the disjunction is true, and hence there is nothing to prove. So we assume that \([ a ] \cap [ b ] \ne \emptyset\) and will show that \([ a ] = [ b ]\text{.}\) Since \([ a ] \cap [ b ] \ne \emptyset\text{,}\) there is an element \(x\) in \(A\) such that

\begin{equation*} x \in [ a ] \cap [ b ]\text{.} \end{equation*}

This means that \(x \in [ a ]\) and \(x \in [ b ]\text{.}\) Consequently, \(x \sim a\) and \(x \sim b\text{,}\) and so we can use the second part of the theorem to conclude that \([ x ] = [ a ]\) and \([ x ] = [ b ]\text{.}\) Hence, \([ a ] = [ b ]\text{,}\) and we have proven that \([ a ] = [ b ]\) or \([ a ] \cap [ b ] = \emptyset\text{.}\)

Theorem 7.3.5 gives the primary properties of equivalence classes. Consequences of these properties will be explored in the exercises. The following table restates the properties in Theorem 7.3.5 and gives a verbal description of each one.

Since this theorem applies to all equivalence relations, it applies to the relation of congruence modulo \(n\) on the integers. Because of the importance of this equivalence relation, these results for congruence modulo \(n\) are given in the following corollary.

Corollary 7.3.7.

Let \(n \in \mathbb{N}\text{.}\) For each \(a \in \mathbb{Z}\text{,}\) let \([ a ]\) represent the congruence class of \(a\) modulo \(n\text{.}\)

  1. For each \(a \in \mathbb{Z}\text{,}\) \(a \in [ a ]\text{.}\)

  2. For each \(a, b \in \mathbb{Z}\text{,}\) \(a \equiv b \pmod n\) if and only if \([ a ] = [ b ]\text{.}\)

  3. For each \(a, b \in \mathbb{Z}\text{,}\) \([ a ] = [ b ]\) or \([ a ] \cap [ b ] = \emptyset\text{.}\)

For the equivalence relation of congruence modulo \(n\text{,}\) Theorem 3.5.9 and Corollary 3.5.10 tell us that each integer is congruent to its remainder when divided by \(n\text{,}\) and that each integer is congruent modulo \(n\) to precisely one of one of the integers \(0, 1, 2, \ldots , n - 1\text{.}\) This means that each integer is in precisely one of the congruence classes \([ 0 ]\text{,}\) \([ 1 ]\text{,}\) \([ 2 ], \ldots , [ {n - 1} ]\text{.}\) Hence, Corollary 7.3.7 gives us the following result.

Corollary 7.3.8.

Let \(n \in \mathbb{N}\text{.}\) For each \(a \in \mathbb{Z}\text{,}\) let \([ a ]\) represent the congruence class of \(a\) modulo \(n\text{.}\)

  1. \(\displaystyle \mathbb{Z} = [ 0 ] \cup [ 1 ] \cup [ 2 ] \cup \cdots \cup [ {n - 1} ]\)

  2. For \(j, k \in \left\{ {0, 1, 2, \ldots , n - 1} \right\}\text{,}\) if \(j \ne k\text{,}\) then \([ j ] \cap [ k ] = \emptyset\text{.}\)

Subsection Partitions and Equivalence Relations

A partition of a set \(A\) is a collection of subsets of \(A\) that “breaks up” the set \(A\) into disjoint subsets. Technically, each pair of distinct subsets in the collection must be disjoint. We then say that the collection of subsets is pairwise disjoint. We introduce the following formal definition.

Definition 7.3.9.

Let \(A\) be a nonempty set, and let \(\mathcal{C}\) be a collection of subsets of \(A\text{.}\) The collection of subsets \(\mathcal{C}\) is a partition of \(\boldsymbol{A}\) provided that

  1. For each \(V \in \mathcal{C}\text{,}\) \(V \ne \emptyset\text{.}\)

  2. For each \(x \in A\text{,}\) there exists a \(V \in \mathcal{C}\) such that \(x \in V\text{.}\)

  3. For every \(V, W \in \mathcal{C}\text{,}\) \(V = W\) or \(V \cap W = \emptyset\text{.}\)

There is a close relation between partitions and equivalence classes since the equivalence classes of an equivalence relation form a partition of the underlying set, as will be proven in Theorem 7.3.10. The proof of this theorem relies on the results in Theorem 7.3.5.

Theorem 7.3.10.

Let \(\sim\) be an equivalence relation on the nonempty set \(A\text{.}\) Then the collection \(\mathcal{C}\) of all equivalence classes determined by \(\sim\) is a partition of the set \(A\text{.}\)

Proof.

Let \(\sim\) be an equivalence relation on the nonempty set \(A\text{,}\) and let \(\mathcal{C}\) be the collection of all equivalence classes determined by \(\sim\text{.}\) That is,

\begin{equation*} \mathcal{C} = \left\{ { {[ a ]} \mid a \in A} \right\}\!\text{.} \end{equation*}

We will use Theorem 7.3.5 to prove that \(\mathcal{C}\) is a partition of \(A\text{.}\)

Item 1 of Theorem 7.3.5 states that for each \(a \in A\text{,}\) \(a \in [ a ]\text{.}\) In terms of the equivalence classes, this means that each equivalence class is nonempty since each element of \(A\) is in its own equivalence class. Consequently, \(\mathcal{C}\text{,}\) the collection of all equivalence classes determined by \(\sim\text{,}\) satisfies the first two conditions of the definition of a partition.

We must now show that the collection \(\mathcal{C}\) of all equivalence classes determined by \(\sim\) satisfies the third condition for being a partition. That is, we need to show that any two equivalence classes are either equal or are disjoint. However, this is exactly the result in Item 3 of Theorem 7.3.5.

Hence, we have proven that the collection \(\mathcal{C}\) of all equivalence classes determined by \(\sim\) is a partition of the set \(A\text{.}\)

Note: Theorem 7.3.10 has shown us that if \(\sim\) is an equivalence relation on a nonempty set \(A\text{,}\) then the collection of the equivalence classes determined by \(\sim\) form a partition of the set \(A\text{.}\)

This process can be reversed. This means that given a partition \(\mathcal{C}\) of a nonempty set \(A\text{,}\) we can define an equivalence relation on \(A\) whose equivalence classes are precisely the subsets of \(A\) that form the partition. This will be explored in Activity 44.

Activity 44. A Partition Defines an Equivalence Relation.

Let \(A = \left\{ {a, b, c, d, e} \right\}\) and let \(\mathcal{C} = \left\{ {\left\{ {a, b, c} \right\}, \left\{ {d, e} \right\}} \right\}\text{.}\)

(a)

Explain why \(\mathcal{C}\) is a partition of \(A\text{.}\)

(b)

Define a relation \(\sim\) on \(A\) as follows: For \(x, y \in A\text{,}\) \(x \sim y\) if and only if there exists a set \(U\) in \(\mathcal{C}\) such that \(x \in U\) and \(y \in U\text{.}\)

Prove that \(\sim\) is an equivalence relation on the set \(A\text{,}\) and then determine all the equivalence classes for \(\sim\text{.}\) How does the collection of all equivalence classes compare to \(\mathcal{C}\text{?}\)

(c)

What we did for the specific partition in Task 44.b can be done for any partition of a set. So to generalize Task 44.b, we let \(A\) be a nonempty set and let \(\mathcal{C}\) be a partition of \(A\text{.}\) We then define a relation \(\sim\) on \(A\) as follows:

For \(x, y \in A\text{,}\) \(x \sim y\) if and only if there exists a set \(U\) in \(\mathcal{C}\) such that \(x \in U\) and \(y \in U\text{.}\)

Prove that \(\sim\) is an equivalence relation on the set \(A\text{.}\)

(d)

Let \(a \in A\) and let \(U \in \mathcal{C}\) such that \(a \in U\text{.}\) Prove that \([ a ] = U\text{.}\)

Activity 45. Equivalence Relations on a Set of Matrices.

The following exercises require a knowledge of elementary linear algebra. We let \(\mathcal{M}_{n, n}\left( \R \right)\) be the set of all \(n\) by \(n\) matrices with real number entries.

(a)

Define a relation \(\sim\) on \(\mathcal{M}_{n, n}( \R )\) as follows: For all \(A, B \in \mathcal{M}_{n, n}( \R )\text{,}\) \(A \sim B\) if and only if there exists an invertible matrix \(P\) in \(\mathcal{M}_{n, n}( \R )\) such that \(B = PAP^{-1}\text{.}\) Is \(\sim\) an equivalence relation on \(\mathcal{M}_{n, n}( \R )\text{?}\) Justify your conclusion.

Define a relation \(R\) on \(\mathcal{M}_{n, n} (\R )\) as follows: For all \(A, B \in \mathcal{M}_{n, n}( \R )\text{,}\) \(A \mathrel{R} B\) if and only if \(\text{ det } (A) = \text{ det } (B)\text{.}\) Is \(R\) an equivalence relation on \(\mathcal{M}_{n, n}( \R )\text{?}\) Justify your conclusion.

(c)

Let \(\sim\) be an equivalence relation on \(\R\text{.}\) Define a relation \(\approx\) on \(\mathcal{M}_{n, n}( \R )\) as follows: For all \(A, B \in \mathcal{M}_{n, n}( \R )\text{,}\) \(A \approx B\) if and only if \(\text{ det } (A) \sim \text{ det } (B)\text{.}\) Is \(\approx\) an equivalence relation on \(\mathcal{M}_{n, n}( \R )\text{?}\) Justify your conclusion.
